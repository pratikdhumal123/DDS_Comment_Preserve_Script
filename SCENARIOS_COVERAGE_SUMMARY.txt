OVERVIEW
--------
This project preserves Confluence inline comments when markdown content is
published to a SCDP/Confluence page. It captures comment markers before
overwrite, re-anchors them after publish, and produces a preservation audit
report with risk classification.

Total scenarios: 47
     - Fully Covered (regression-proven): 40
  - Partially Covered (manual review recommended): 6
     - Not Covered / By Design: 1

================================================================================
OUTCOME CLASSES LEGEND
================================================================================
Exact preserve           — Comment stays on the same logical anchor text
Edited-text preserve     — Comment moves to the rewritten local replacement
Local fallback preserve  — Comment preserved near original using safe nearby anchor
Recoverable heading      — Comment preserved on nearest heading or section anchor
preserve                   when direct text anchoring is no longer safe
Audit-only               — Cannot safely re-attach; reported for manual review

================================================================================
STRICT FLAGS (for production use)
================================================================================
--require-visible-inline-markers   Fail (exit 2) if any active comment becomes
                                   invisible after publish
--require-low-risk-reanchor        Fail (exit 3) if risk assessment marks the
                                   run as requiring manual review
